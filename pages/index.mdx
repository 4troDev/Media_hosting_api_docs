# Media Hosting API Documentation

## Introduction

Welcome to the documentation for the Media Hosting API, an open-source solution designed to streamline media file hosting and management. This API provides a straightforward and efficient way to handle various types of media files, allowing you to seamlessly integrate media file management into your Node.js applications.

---

## Endpoints

The Media Hosting API exposes the following endpoints:

- **Check Media File Existence**:
  - Endpoint: `GET /api/v1/check/:id`
  - Description: Check if a media file exists by providing its ID.

- **Delete Media File**:
  - Endpoint: `DELETE /api/v1/delete/:id`
  - Description: Remove a media file by providing its ID.

- **Stream Media (Videos or MP3)**:
  - Endpoint: `GET /api/v1/play/:id`
  - Description: Stream videos or MP3 files by providing the media file ID.

- **Upload Media File**:
  - Endpoint: `POST /api/v1/upload`
  - Description: Upload media files, including images, video files, and audio files.

- **View Media Source Link**:
  - Endpoint: `GET /api/v1/view/:id`
  - Description: Retrieve the source link of the media file by providing its ID.

## Usage

To interact with the Media Hosting API, you can make HTTP requests to the specified endpoints. Below are examples of key functionalities:

### Check Media File Existence

```javascript
const result = mediaHostingApi.checkMediaFileExistence('123456');
console.log(result);
```

### Delete Media File

```javascript
const result = mediaHostingApi.deleteMediaFile('123456');
console.log(result);
```

### Stream Media (Videos or MP3)

```javascript
const result = mediaHostingApi.streamMedia('123456');
console.log(result);
```

### Upload Media File

```javascript
const result = mediaHostingApi.uploadMediaFile('/path/to/file.jpg');
console.log(result);
```

### View Media Source Link

```javascript
const result = mediaHostingApi.viewMediaSourceLink('789012');
console.log(result);
```

### Accessing Errors

If you encounter any errors while using the API, you can access the specific error message by checking the error property in the response.
